# tests/security — воспроизведение находок аудита

Артефакты состязательного QA/security-аудита (см. корневые `TESTMAP.md` и `BUGREPORT.md`).

## Файлы
- **`authz-rls-probe.mjs`** — Node-скрипт (без зависимостей, нужен Node 18+ с global `fetch`).
  Воспроизводит проверки авторизации/RLS: аноним заблокирован, открытость регистрации,
  чтение всех заказов пользователем без профиля, запись/переход/спуф не-админом, и
  негативные проверки (delete / самоэскалация / запись цен должны быть закрыты).

## Правила безопасности (обязательно)
- **Не запускать пишущие проверки по боевой базе.** Пишущая часть (D–F) активируется только
  `CONFIRM_WRITES=1` и создаёт собственный заказ с префиксом `[QA-SEC-TEST]`.
- Предпочтительная среда — **Supabase preview-ветка** или **отдельный тест-аккаунт**.
- Боевые данные не трогать. За собой убирать (скрипт печатает SQL для удаления тест-заказа;
  полное удаление заказа — только под admin, т.к. `delete` в RLS = `is_admin()`).

## Запуск

Только конфиг-проверки (безопасно, без записи, без пользователя):
```bash
SB_URL=https://<ref>.supabase.co \
SB_ANON=<anon-key> \
node authz-rls-probe.mjs
```

Полный сценарий (нужен ПОДТВЕРЖДённый тест-пользователь; писать только на preview/тест):
```bash
SB_URL=https://<ref>.supabase.co SB_ANON=<anon-key> \
TEST_EMAIL=<confirmed-test-user> TEST_PASS=<pass> \
CONFIRM_WRITES=1 node authz-rls-probe.mjs
```

Подтверждение почты тест-пользователя (это шаг злоумышленника, не уязвимость) — в Dashboard
или SQL на preview:
```sql
update auth.users set email_confirmed_at = now() where email = '<TEST_EMAIL>';
```

## Что показывает эталонный прогон (прод-схема, 2026-07-22)
- Аноним: `erp_orders` → 0 строк (RLS требует `authenticated`) — защищено.
- GoTrue: `disable_signup=false`, `mailer_autoconfirm=false` — регистрация открыта, нужен confirm.
- Пользователь без профиля/approve: видит **все 146** заказов — утечка (HI-2/CR-1).
- Не-админ: создаёт заказ, PATCH-ит (`priority=999`), ставит `done_early`+`shipped` — обход (HI-1/HI-3).
- Комментарий сохраняется с автором «Директор (СПУФ)» — спуф (ME-2).
- Негативные (защита держится): delete → 0 строк; profiles self-escalation → 403;
  запись `app_config.prices` → 403.

> Скрипт `.mjs` НЕ подхватывается Vitest (нет суффикса `.test.`) — запускается вручную,
> в CI не участвует.
