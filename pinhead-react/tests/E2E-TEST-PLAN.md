# Pinhead Order Studio — E2E Test Plan

**URL:** https://pinhead-os.vercel.app  
**Дата:** 07.04.2026  
**Тестов:** 56 сценариев в 8 группах  
**Стек:** Playwright + Vitest

---

## Группа 1 — Авторизация (11 тестов)

### 1.1 Успешный логин
- Открыть приложение → форма входа → ввести email/пароль → нажать «Войти»
- Ожидание: авторизация, хедер с навигацией, визард на шаге 0

### 1.2 Неверные credentials
- Ввести несуществующий email → «Войти»
- Ожидание: ошибка в `.auth-error`, остаёмся на форме

### 1.3 Пустые поля
- Нажать «Войти» без заполнения
- Ожидание: ничего не происходит, нет краша

### 1.4 Регистрация
- Переключить на «Регистрация» → заполнить имя/email/пароль → отправить
- Ожидание: экран «Ожидание подтверждения»

### 1.5 Регистрация с пустыми полями
- Отправить форму регистрации с пропущенными полями
- Ожидание: форма не отправляется

### 1.6 Неодобренный аккаунт
- Войти с approved: false
- Ожидание: экран ожидания подтверждения, кнопка «Выйти»

### 1.7 Выход
- Нажать «Выйти» в хедере
- Ожидание: возврат к форме входа

### 1.8 Навигация для admin
- Все разделы доступны: Express, Заказы, SKU, Цены, Аналитика, Admin

### 1.9 Навигация для manager
- /prices, /sku, /admin, /analytics → редирект на /

### 1.10 Навигация для production
- Видит только approved/production заказы, аналитика доступна

### 1.11 Навигация для designer
- Нет Express, нет аналитики

## Группа 2 — Визард заказа (11 тестов)

### 2.1 Полный happy path (футболка + шелкография)
- Шаг 0: SKU → ткань → цвет → размеры M:10, L:20
- Шаг 1: зона «Грудь» → шелкография A3 2 цвета → путь к макетам
- Шаг 2: проверить позицию добавилась
- Шаг 3: имя, телефон, дедлайн
- Шаг 4: проверить разбивку → «Сохранить заказ» → PH-XXXX

### 2.2 Автосохранение черновика
- Заполнить шаг 0 → F5 → данные восстановлены

### 2.3 Блокировка навигации
- На шаге 1+ нажать «Заказы» → диалог «Заказ не сохранён»
- «Остаться» → остаёмся, «Всё равно уйти» → переход

### 2.4 Предупреждение шелкография < 50 шт
- Тираж 10, зона front, screen → предупреждение + чекбокс

### 2.5 Без нанесения
- Нажать «Без нанесения» → зоны скрыты → «Далее» активна

### 2.6 Несколько техник в одном заказе
- front: шелко, back: DTG, hood: вышивка

### 2.7 Кастомные размеры
- Добавить XXS → проверить → удалить

### 2.8 Валидация на итоге
- Без имени клиента → ошибки валидации → «Сохранить» disabled

### 2.9 Шаблоны
- Сохранить шаблон → новый заказ → применить → удалить

### 2.10 Обработки (extras)
- Выбрать люверсы + шнурок → цена увеличилась

### 2.11 Print Preview
- /print → ТЗ с данными заказа → кнопки «ПЕЧАТЬ» и «СКАЧАТЬ PDF»

## Группа 3 — Kanban (17 тестов)

### 3.1 Загрузка доски
- 5 колонок, карточки по статусам, стат-бар

### 3.2 Поиск и фильтр
- Поиск по имени/номеру/Bitrix → фильтр по типу изделия

### 3.3 Открыть OrderDrawer
- Клик по карточке → drawer с деталями

### 3.4 Закрыть drawer
- ✕, клик по фону, Escape

### 3.5 Смена статуса через drawer
- Нажать кнопку статуса → toast → карточка переместилась

### 3.6 Смена статуса хоткеями (1-5)
- В открытом drawer нажать 2 → статус «На проверке»

### 3.7 Смена статуса через меню •••
- Клик по ••• → выбрать статус

### 3.8 Drag & Drop
- Перетащить карточку → статус «Готов» требует confirm

### 3.9 Дублировать (drawer)
- «Дублировать» → переход в визард → toast

### 3.10 Дублировать (кнопка на карточке)
- ⎘ → аналогично 3.9

### 3.11 Удалить с confirm
- ✕ → confirm dialog → отмена/подтверждение

### 3.12 Комментарии
- Ввести текст → → отправить → Enter отправляет

### 3.13 Хоткеи Kanban
- ? → справка, / → поиск, n → новый заказ

### 3.14 Открыть ТЗ
- «Открыть ТЗ» → /print

### 3.15 Пагинация
- «Загрузить ещё» → подгрузка 50 заказов

### 3.16 Фильтр по роли production
- Только approved + production заказы

### 3.17 Дедлайн-бейджи
- Просрочен (красный), 2 дня (оранжевый), 5 дней (жёлтый)
