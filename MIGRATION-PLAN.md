# План миграции Pinhead OS на React

## Принципы безопасности
- **Каждый шаг — отдельный коммит.** Можно откатить `git revert`
- **Ванильная версия остаётся рабочей** до полного перехода (параллельная разработка)
- **После каждого шага — проверка** вместе с тобой
- **Данные (SKU, ткани, цены) не трогаем** — переносим как есть

---

## Шаг 1: Инициализация React-проекта (Vite)
**Что делаем:**
- `npm create vite@latest pinhead-react -- --template react`
- Настраиваем структуру папок
- Копируем CSS-переменные из `css/base.css`
- Переносим Google Fonts

**Результат:** Пустое React-приложение с нашими цветами и шрифтами
**Проверка:** Открывается dev-сервер, видны наши шрифты и цвета
**Риск:** Нулевой — создаём в отдельной папке

---

## Шаг 2: Перенос данных (без логики)
**Что делаем:**
- Копируем `SKU_CATALOG`, `FABRICS_CATALOG`, `COLOR_PALETTES`, `PRICES`, `EXTRAS_CATALOG`, `LABEL_CONFIG` → `src/data/`
- Экспортируем как ES-модули
- Копируем константы: `TYPE_NAMES`, `TECH_NAMES`, `ZONE_LABELS`, `FABRIC_NAMES`

**Результат:** Все справочники доступны через `import`
**Проверка:** `console.log(SKU_CATALOG.length)` в App.jsx — число совпадает с оригиналом
**Риск:** Нулевой — только копирование данных

---

## Шаг 3: State management (Zustand)
**Что делаем:**
- Устанавливаем Zustand (минималистичный, 1 КБ)
- Создаём `src/store/orderStore.js` — все 40+ полей из `state`
- Создаём `src/store/uiStore.js` — текущий шаг, модалки, фильтры
- Добавляем actions: `setType()`, `setFabric()`, `setQty()`, `goToStep()` и т.д.

**Результат:** Централизованный state с иммутабельными обновлениями
**Проверка:** React DevTools → store содержит все поля, actions работают
**Риск:** Низкий — state пока ни к чему не привязан

---

## Шаг 4: Layout + навигация (каркас)
**Что делаем:**
- `<App>` → `<Header>` + `<StepProgress>` + `<StepPanel>` + `<Footer>`
- 6 шагов как компоненты-заглушки: `<StepProduct/>`, `<StepExtras/>`, `<StepDesign/>`, `<StepClient/>`, `<StepSummary/>`, `<StepConfirm/>`
- Навигация: кнопки «Назад/Далее», прогресс-бар
- Переносим CSS из `layout.css`

**Результат:** Каркас приложения с переключением шагов
**Проверка:** Клик по шагам — переключаются панели, прогресс-бар обновляется
**Риск:** Низкий

---

## Шаг 5: Step 0 — Выбор изделия
**Что делаем:**
- `<GarmentSelector>` — список SKU по категориям (из SKU_CATALOG)
- `<FabricSelector>` — сетка тканей, фильтрация по категории
- `<ColorPalette>` — свотчи цветов (Medastex + CottonProm)
- `<SizeTable>` — ввод количества по размерам
- `<FitSelector>` — regular/oversized
- Привязка к store

**Результат:** Полностью рабочий первый шаг
**Проверка:** Выбрать SKU → ткань отфильтровалась → цвет выбран → размеры введены → store обновился
**Риск:** Средний — самый большой шаг, много UI-компонентов

---

## Шаг 6: Step 1 — Обработки (Extras)
**Что делаем:**
- `<ExtrasGrid>` — карточки обработок с SVG-иконками
- Привязка к store (toggle выбранных)

**Результат:** Карточки обработок кликабельны, цена обновляется
**Проверка:** Выбрать обработки → они сохранены в store
**Риск:** Низкий

---

## Шаг 7: Step 2 — Дизайн (зоны + бирки)
**Что делаем:**
- `<ZoneSelector>` — динамические зоны по типу изделия
- `<TechSelector>` — выбор техники для каждой зоны
- `<LabelConfigurator>` — 3 аккордеон-секции (Care/Main/Hang)
- `<FileUpload>` — drag & drop загрузка дизайна
- Перенос логики из `zones.js`, `ui-design.js`

**Результат:** Полностью рабочий шаг дизайна
**Проверка:** Выбрать зоны → техники → загрузить файл → конфигурировать бирки
**Риск:** Средний — сложная логика зон/техник

---

## Шаг 8: Движок ценообразования
**Что делаем:**
- `src/hooks/usePricing.js` — хук с полной логикой из `pricing.js`
- `screenLookup()`, `calcTotal()`, сурчарджи, матрицы
- `<PriceDisplay>` — компонент с итогом
- Автопересчёт при любом изменении store

**Результат:** Цены считаются реактивно
**Проверка:** Менять SKU/ткань/тираж/технику → цена обновляется мгновенно, совпадает с оригиналом
**Риск:** Высокий — критичная бизнес-логика, нужна сверка с оригиналом

---

## Шаг 9: Step 3 — Клиент + Step 4 — Сводка
**Что делаем:**
- `<ClientForm>` — имя, контакт, email, телефон, мессенджер, дедлайн, адрес, заметки
- `<OrderSummary>` — полная сводка заказа (аналог `buildSummary()`)
- `<MockupPreview>` — SVG-мокап изделия

**Результат:** Заполнение клиента + просмотр итога
**Проверка:** Заполнить форму → сводка показывает все данные корректно
**Риск:** Низкий

---

## Шаг 10: Валидация шагов
**Что делаем:**
- `src/hooks/useValidation.js` — логика из `validateStep()`
- Блокировка кнопки «Далее» если шаг не валиден
- Progressive sections: секции заблокированы пока не выбраны зависимости
- Сообщения об ошибках

**Результат:** Нельзя перейти на следующий шаг без заполнения обязательных полей
**Проверка:** Попробовать перейти без выбора SKU → блок, выбрать SKU → разблокировано
**Риск:** Низкий

---

## Шаг 11: Автосохранение (Draft)
**Что делаем:**
- `src/hooks/useDraft.js` — debounced сохранение store в localStorage
- Восстановление при загрузке
- Кнопка «Сбросить черновик»

**Результат:** Перезагрузка страницы → данные на месте
**Проверка:** Заполнить форму → перезагрузить → всё восстановилось
**Риск:** Низкий

---

## Шаг 12: Supabase — авторизация
**Что делаем:**
- `src/hooks/useAuth.js` — login, register, logout, checkApproved
- `<AuthScreen>` — экран входа/регистрации
- `<AdminPanel>` — управление пользователями
- Supabase SDK через npm вместо CDN

**Результат:** Вход/регистрация/выход работают
**Проверка:** Залогиниться → увидеть свои заказы → разлогиниться
**Риск:** Средний — нужно проверить совместимость с существующей Supabase базой

---

## Шаг 13: История заказов + Канбан
**Что делаем:**
- `<OrderHistory>` — список заказов
- `<KanbanBoard>` — 5 колонок, drag & drop (react-beautiful-dnd или dnd-kit)
- `<KanbanCard>` — карточка заказа
- Фильтры, статистика, Excel-экспорт
- Сохранение/загрузка/дублирование заказов

**Результат:** Полноценная канбан-доска
**Проверка:** Создать заказ → появился в канбане → перетащить → статус изменился
**Риск:** Средний

---

## Шаг 14: Print Preview
**Что делаем:**
- `<PrintPreview>` — HTML-документ ТЗ
- Все секции: шапка, изделие, дизайн, клиент, итого
- Кнопки «Печать/PDF» и «Закрыть»
- Инлайн-стили для корректной печати

**Результат:** ТЗ генерируется и печатается
**Проверка:** Открыть preview → все данные на месте → Ctrl+P работает
**Риск:** Средний — инлайн-CSS для печати

---

## Шаг 15: Express Calculator + Price Editor
**Что делаем:**
- `<ExpressCalc>` — быстрый расчёт
- `<PriceEditor>` — редактор цен (полноэкранный)
- `<SkuCatalog>` — редактор SKU

**Результат:** Все вспомогательные инструменты работают
**Проверка:** Открыть экспресс-калькулятор → посчитать → совпадает с оригиналом
**Риск:** Средний

---

## Шаг 16: Agentation
**Что делаем:**
- `npm install agentation -D`
- Добавляем `<Agentation />` в dev-режиме
- Настраиваем MCP-сервер (опционально)

**Результат:** Можно кликать по элементам → получать контекст для ИИ
**Проверка:** Активировать → кликнуть по кнопке → увидеть селектор и метаданные
**Риск:** Нулевой — только dev-зависимость

---

## Шаг 17: Финализация
**Что делаем:**
- Удаляем старую ванильную версию (или оставляем как fallback)
- Билд: `npm run build` → `dist/`
- Деплой
- Финальное тестирование всех сценариев

**Результат:** Продакшн-готовое React-приложение
**Проверка:** Полный прогон: создание заказа от начала до конца
**Риск:** Низкий (если все предыдущие шаги пройдены)

## Структура папок React-проекта

```
pinhead-react/
├── public/
├── src/
│   ├── components/
│   │   ├── layout/        (Header, StepProgress, Footer)
│   │   ├── steps/         (StepProduct, StepExtras, StepDesign, StepClient, StepSummary)
│   │   ├── garments/      (GarmentSelector, FabricSelector, ColorPalette, SizeTable)
│   │   ├── design/        (ZoneSelector, TechSelector, LabelConfigurator, FileUpload)
│   │   ├── orders/        (KanbanBoard, KanbanCard, OrderHistory)
│   │   ├── editors/       (ExpressCalc, PriceEditor, SkuCatalog)
│   │   ├── output/        (PrintPreview, OrderSummary, MockupPreview)
│   │   └── auth/          (AuthScreen, AdminPanel)
│   ├── data/              (SKU_CATALOG, FABRICS, COLORS, PRICES, EXTRAS, LABELS)
│   ├── store/             (orderStore, uiStore)
│   ├── hooks/             (usePricing, useValidation, useDraft, useAuth)
│   ├── styles/            (CSS файлы)
│   ├── App.jsx
│   └── main.jsx
├── package.json
└── vite.config.js
```
