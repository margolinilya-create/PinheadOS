# Deep Research: Полный аудит PinheadOS

> Дата: 2026-03-20
> Метод: 5 параллельных Explore-агентов (структура, качество кода, тесты/CI, безопасность/перформанс, бизнес-логика)

---

## Что такое PinheadOS

Конфигуратор заказов на пошив одежды с нанесением. React 19 SPA + Supabase PostgreSQL, деплой на Vercel.

**Масштаб:** ~8,000 строк продакшн-кода, ~5,300 строк тестов, 21 компонент, 4 Zustand стора, 11 CSS файлов.

## 2. Критические находки

### 2.1 Отсутствие типизации
- Чистый JavaScript, нет TypeScript, нет PropTypes
- Zustand сторы без типов — рефакторинг рискован
- Данные из Supabase не валидируются
- **Риск:** рантайм-ошибки при изменении структуры данных

### 2.2 Минимальная обработка ошибок
- Нет Error Boundary — краш React = белый экран
- PriceEditor.jsx бросает ошибки без catch
- Supabase-вызовы часто с пустым catch
- Toast-система есть, но используется редко для ошибок
- **Риск:** пользователь не понимает что пошло не так

### 2.3 Accessibility = 3/10
- Ноль `aria-label`, `role`, `aria-live`
- Иконочные кнопки без текста (⎘, ✕)
- Декоративный SVG не скрыт от screen reader
- Формы без связанных `<label>`
- **Риск:** недоступность для пользователей с ограничениями

### 2.4 Монолитный стор
- `useStore.js` = 486 строк, управляет всем: навигация, SKU, ткани, цвета, размеры, зоны, лейблы, extras, multi-items, драфт
- Сложные селекторы по 14 строк
- **Риск:** сложность поддержки, когнитивная нагрузка

### 2.5 Тестовое покрытие — 53%
- **Без тестов:** StepGarment, StepDesign, StepExtras, StepSummary, ZoneMockup, ExpressCalc, PriceEditor, useStore, useOrdersStore, pricing.js, catalogs.js
- Критический бизнес-код (pricing, main store, summary) не покрыт
- **Риск:** регрессии при изменениях

## 3. Средние проблемы

### 3.1 CSS без scoping
- Глобальные CSS классы, без CSS Modules / Tailwind
- 56 inline-стилей по кодовой базе
- BEM-подобное именование, но без гарантий уникальности
- Потенциальные конфликты специфичности

### 3.2 Дублирование кода
- Zustand selector-паттерны повторяются в каждом компоненте
- localStorage-операции разбросаны (PriceEditor, useDraft, StepGarment)
- Zone parameter buttons дублируются в 3 компонентах
- Нет централизованного storage-утилита

### 3.3 Безопасность — 8/10
- Supabase publishable key захардкожен (допустимо, но лучше в env)
- Нет input sanitization для текстовых полей (name, notes)
- Нет session timeout (24h по умолчанию Supabase)
- DEV_MODE bypass в продакшн-коде
- XSS через dangerouslySetInnerHTML — LOW risk (SVG из trusted source)

### 3.4 Performance
- Нет React.memo() — ни одного использования
- useCallback минимально (только KanbanBoard)
- SKU reorder без debounce → множественные записи в БД
- Нет пагинации (лимит 200 заказов)
- select('*') вместо выборочных полей

## 5. UX-проблемы

1. **Редактирование item** — возврат на шаг 0, а не на нужный шаг
2. **Нет breakdown цены** — пользователь видит итог, но не составные части
3. **Зоны техники** — 5 типов с разными параметрами, нет пояснений
4. **Keyboard shortcuts в Kanban** — нет визуальной подсказки
5. **Дедлайн** — только цветной badge, нет блокировки/оповещения
6. **Валидация** — непоследовательная (где-то inline, где-то по клику)

## Рекомендация

Проект функционально зрелый — основные фичи работают. Главные зоны роста:
