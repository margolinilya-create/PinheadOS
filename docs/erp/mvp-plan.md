# ERP — MVP: план, доменная модель, промты

> Источник требований: `docs/erp/discovery-questions.md` (71 вопрос, заполнено заказчиком).
> Google-таблица «вся логика» — поступит позже, детализирует Фазы 3–5 и отложенные блоки.

## 1. Суть MVP

**Прозрачность производства + онлайн-статусы во всех цехах.**
Заказ → изделия → маршрут по цехам → статусы по этапам → трекинг «где заказ» → дашборд загрузки/узких мест/сроков/алертов.
Цель: ускорить внутренние процессы, чтобы информацию видели все руководители.

### Зафиксированные решения
- Order Studio (ТЗ/цены) — **заархивировать за feature-flag** (не удалять).
- Новый ERP — **с чистого листа**.
- Supabase — **`pinhead-os-v2`**; старый ERP (redesign/v2) — **зачистить**, отделив ERP-таблицы от создания заказов.
- Гранулярность — **гибрид: заказ + дробление партий**.
- Маршрут — **вручную при создании заказа**.
- Сроки — **дедлайн заказа + плановые даты по этапам** (цветовой план/факт).
- Статус — **меняет любой сотрудник цеха**, бригадир подтверждает / отмечает «невозможно» + комментарий.

## 2. Цеха / участки (11)

Закупка · Логистика · Экспериментальный · Закройный · Швейный · Шелкография · ДТФ · Вышивка · Склад · Склад ГП · ВТО.

Поток **нелинейный**: заказ может проходить любой набор этапов (напр. только брендирование); партия дробится и идёт параллельно; есть возвраты по браку; нужен учёт НЗП.

## 3. Роли и доступ

| Роль | Доступ |
|---|---|
| Директор | Всё |
| Диспетчер / рук. производства | Все заказы и цеха, очередь, приоритеты, маршруты |
| Начальник цеха / бригадир | Очередь своего цеха, подтверждение, блокировки+коммент |
| Сотрудник цеха *(позже)* | Самоотметка выработки |
| Менеджер | Создание заказов, статусы своих заказов |
| Закупщик | Заказ ткани, поставщики, приёмка, расход на заказ |
| Кладовщик | Приход/расход, склад ГП |
| HR | Управление сотрудниками |

## 4. Доменная модель (MVP)

```
departments        участки/цеха (seed 11): code, name, type, order_index, active
employees          сотрудник: name, login, role, department_id, extra_departments[],
                   is_foreman, active   (CRUD: HR + Директор)
orders             заказ: number, title, customer, priority, deadline,
                   status, created_by, notes
order_items        изделие: order_id, name/model, qty, sizes(jsonb), color, notes
batches            под-партия изделия (для дробления): order_item_id, qty, label
stage_tasks        этап в цехе: batch_id, department_id, seq, planned_date,
                   status(pending|in_progress|done|blocked|skipped|rework),
                   started_at, finished_at, qty_done, comment
stage_events       аудит смены статуса: stage_task_id, actor, from→to, comment, at
                   + подтверждение бригадира (confirmed_by, confirmed_at)
defects            брак/возврат: stage_task_id, qty, reason, return_to_department_id
suppliers          поставщик (lite)
fabric_orders      закупка ткани под заказ: order_id, supplier_id, fabric, qty, unit,
                   cost, received(приёмка), consumption(расход на заказ)
```

**Маршрут** = упорядоченный список `stage_tasks` на каждую `batch`. Диспетчер/менеджер при создании заказа выбирает цеха и порядок → генерируются `stage_tasks` с плановыми датами. Возврат по браку создаёт `rework`-задачу на нужный цех.

## 5. Экраны

- **Производственная доска** — по цехам (колонка = очередь `stage_tasks` цеха); смена статуса, блокировка+коммент, подтверждение бригадиром.
- **Трекинг заказа** — карточка: прогресс по этапам «как трекинг посылки», план/факт, брак/возвраты.
- **Дашборд руководителя** — где заказы, загрузка цехов, узкие места, план/факт сроков, алерты (просрочка/простой/брак).
- **Заказы** — список + создание (вручную / визард-лайт), изделия, маршрут, приоритет, дедлайн, плановые даты.
- **Сотрудники** (HR/Директор), **Закупка ткани** (закупщик).

## 6. Поэтапный план

- **Фаза 0 — Подготовка** *(не зависит от таблицы)*
  - Архивация Order Studio за `feature.orderStudio` (роуты/бандл скрыты, код и таблицы целы).
  - Зачистка старого ERP в `pinhead-os-v2` (с явным подтверждением перед удалением).
  - Каркас нового ERP: роутинг, layout, роли/guards, auth.
- **Фаза 1 — Справочники ядра:** цеха (seed 11), сотрудники (CRUD HR/Директор), роли/доступы, RLS.
- **Фаза 2 — Заказы:** создание (вручную/визард-лайт), изделия, ручной маршрут по цехам, приоритет, дедлайн + плановые даты по этапам, дробление партий.
- **Фаза 3 — Производственная доска:** очереди по цехам, статусы (любой в цехе + подтверждение бригадира), блокировки+коммент, брак/возвраты, НЗП.
- **Фаза 4 — Прозрачность:** трекинг заказа, дашборд руководителя, загрузка/узкие места/план-факт, алерты.
- **Фаза 5 — Ткань (лайт):** закупка под заказ, поставщики, приёмка, расход на заказ.

### В ящике (позже, по списку заказчика)
Оплата труда · детали Раскроя/Пошива/ВТО · Упаковка · фурнитура/расходники · склад-инвентаризация · табель рабочего времени · печатные документы (маршрутный лист, наряд, этикетка) · Bitrix24 · Честный знак.

## 7. Промты для сборки (Фазы 0–2 готовы, 3–5 уточним после таблицы)

**P0.1 — Архивация Order Studio.** Спрятать роуты `/`, `/express`, `/print`, `/sku`, `/prices`, `/analytics` визарда за флаг `feature.orderStudio` (по умолчанию off). Код и тесты не удалять, из дефолтного бандла убрать (lazy + guard). Корень приложения → новый ERP-layout. Откат = флаг on.

**P0.2 — Зачистка ERP-схемы.** В `pinhead-os-v2`: показать список таблиц redesign/v2 ERP, отделить их от таблиц создания заказов, подготовить миграцию drop с бэкапом-дампом. НЕ применять без подтверждения заказчика.

**P0.3 — Каркас ERP.** React-роутинг новых экранов, layout с навигацией по ролям, guards по ролям, seed справочника `departments` (11). Zustand-сторы: `useDepartmentsStore`, `useEmployeesStore`.

**P1.1 — Сотрудники.** CRUD сотрудников (HR/Директор): логин, роль, цех(а), бригадир-флаг, soft-delete. RLS по ролям.

**P2.1 — Заказы + изделия.** Создание заказа (вручную/визард-лайт), изделия (qty, размеры, цвет), приоритет, дедлайн.

**P2.2 — Маршрут + плановые даты.** UI выбора цехов и порядка при создании → генерация `stage_tasks`; дробление партий; плановая дата на этап.

*(P3.x производственная доска, P4.x прозрачность, P5.x ткань — детализируем после Google-таблицы.)*
