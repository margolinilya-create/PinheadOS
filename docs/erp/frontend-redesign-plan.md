## Цель

Единая современная ERP в духе Linear/Plane/ClickUp/Odoo 18: компактно, светлый фон, белые
рабочие области, минимум цвета (только статусы/ошибки/просрочки/главные действия), единые
переиспользуемые компоненты во всех разделах.

## Что уже есть (база, переиспользуем)

Токен-система (`src/index.css`), тёмная тема через токены; `.table`/`.tableWrap`; набор чипов
`.chip*` + лейбл-мапы в `erp/types.ts`; `SearchInput`; модалка (`.modal*`); глобальные Toast +
imperative Confirm; скелетоны (`ErpSkeletons`); `InlineEdit`; KPI-плитки; компактная канбан-карточка.

## Чего не хватает (собственно редизайн)

> ✅ Историческое (исходный разбор перед стартом). Всё ниже реализовано — см. «Статус» вверху:
> `Drawer`/`Pipeline`/`Stepper`/`FilterBar`/`Pagination` в `erp/components/`, счётчики в навигации,
> пагинация таблиц. `flowStepper`-легенда «Подряда» заменена нумерованным `Stepper` (PR6).

## Фазовый план (последующими PR) — ✅ выполнен (PR1–PR6, сессия 20; хвосты — сессия 21)

1. **Дизайн-примитивы:** `Drawer`, `FilterBar`, `Pagination`, `EmptyState`-компонент,
   flow-`Stepper`, `Pipeline` — в `erp/components/`, на существующих токенах/классах.
2. **Навигация:** счётчики активных задач по каждому разделу (хелперы `orderHelpers.ts`
   `readyCountFor`/`overdueUnackCountFor` уже есть — расширить на все разделы).
3. **Производство — Kanban:** компактная карточка (№/название/изделие/кол-во/срок/просрочка/
   ответственный/главное действие); всё остальное — в `Drawer`.
4. **Закупка / Склад:** компактные таблицы + детали в `Drawer` (частично уже таблицы).
5. **Подряд:** таблица + Stepper (базово сделано в 4.2.4) → довести до единого `FilterBar`/`Drawer`.
6. **Экспериментальный цех:** `Pipeline` (Лекала→Закупка→Проработка→Нанесение→Примерка→Готово)
   с бейджем текущей локации.
7. **Единая карточка заказа** (Drawer) с вкладками: Информация/Материалы/ТЗ/Комментарии/История/
   Файлы/Маршрут — одна на все разделы.
