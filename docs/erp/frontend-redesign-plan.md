# Frontend ERP Pinhead — план редизайна (отдельный трек)

> Зафиксировано в сессии 19 (2026-07-22). Ведётся по **макетам заказчика + UI Kit** (сессия 20).
> Backend/API/маршруты/бизнес-логику/статусы НЕ менять — изменяется только frontend.

## Статус

- **PR1 — фундамент/оболочка ✅ (сессия 20):** ERP-scoped палитра из UI Kit (primary `#2563EB`,
  статусные цвета, поверхности) на корне `.shell` — Order Studio не затронут, тёмная тема сохранена;
  вертикальный сгруппированный `Sidebar` (`erp/layout/Sidebar.jsx`) со счётчиками активных задач и
  сворачиванием; обновлённый Header; компонент `Badge` (`erp/components/Badge.jsx`) + перекраска
  `.chip*` под палитру (в работе=синий, ожидает=амбер); count-хелперы в `orderHelpers.ts`.
- **Дальше:** PR2 Dashboard → PR3 таблицы Склад/Закупка + FilterBar/Pagination → PR4 карточка-Drawer
  со Stepper → PR5 Подряд Stepper + Эксперимент Pipeline → PR6 добивка единых компонентов.

## Цель

Единая современная ERP в духе Linear/Plane/ClickUp/Odoo 18: компактно, светлый фон, белые
рабочие области, минимум цвета (только статусы/ошибки/просрочки/главные действия), единые
переиспользуемые компоненты во всех разделах.

## Что уже есть (база, переиспользуем)

Токен-система (`src/index.css`), тёмная тема через токены; `.table`/`.tableWrap`; набор чипов
`.chip*` + лейбл-мапы в `erp/types.ts`; `SearchInput`; модалка (`.modal*`); глобальные Toast +
imperative Confirm; скелетоны (`ErpSkeletons`); `InlineEdit`; KPI-плитки; компактная канбан-карточка.

## Чего не хватает (собственно редизайн)

| Примитив | Статус | Где нужен |
|---|---|---|
| Переиспользуемый **Drawer** (боковая панель деталей) | нет в ERP (детали открываются как маршрут `/orders/:id`; есть только легаси `OrderDrawer` в Order Studio) | карточка заказа во всех разделах |
| **Pipeline** (воронка) | нет | Экспериментальный цех |
| flow-**Stepper** | нет (есть read-only `StageStepper` этапов; в сессии 19 добавлена лёгкая `flowStepper`-легенда в «Подряд») | Подряд, статусные потоки |
| единый **FilterBar** | нет (каждый экран катает свой `.toolbar`) | все разделы |
| **счётчики активных задач** в навигации по разделам | только `/queue` | вся верхняя навигация |
| **пагинация** таблиц | нет | Закупка/Склад/Подряд/Заказы |

## Фазовый план (последующими PR)

1. **Дизайн-примитивы:** `Drawer`, `FilterBar`, `Pagination`, `EmptyState`-компонент,
   flow-`Stepper`, `Pipeline` — в `erp/components/`, на существующих токенах/классах.
2. **Навигация:** счётчики активных задач по каждому разделу (хелперы `orderHelpers.ts`
   `readyCountFor`/`overdueUnackCountFor` уже есть — расширить на все разделы).
3. **Производство — Kanban:** компактная карточка (№/название/изделие/кол-во/срок/просрочка/
   ответственный/главное действие); всё остальное — в `Drawer`.
4. **Закупка / Склад:** компактные таблицы + детали в `Drawer` (частично уже таблицы).
5. **Подряд:** таблица + Stepper (базово сделано в 4.2.4) → довести до единого `FilterBar`/`Drawer`.
6. **Экспериментальный цех:** `Pipeline` (Лекала→Закупка→Проработка→Нанесение→Примерка→Готово)
   с бейджем текущей локации.
7. **Единая карточка заказа** (Drawer) с вкладками: Информация/Материалы/ТЗ/Комментарии/История/
   Файлы/Маршрут — одна на все разделы.

## Не менять

backend, API, маршруты заказов, бизнес-логику, переходы между этапами, систему статусов.
