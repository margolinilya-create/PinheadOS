# Systematic Debugging — Pinhead Order Studio

Следуй этим шагам строго по порядку. Не переходи к следующему шагу не завершив текущий.

## Шаг 1 — Воспроизведи баг

Прежде чем что-либо менять:
- Прочитай описание бага от пользователя
- Найди в коде точное место где происходит ошибка
- Сформулируй: «Баг воспроизводится когда X — ожидается Y — получается Z»

Файлы для первичного поиска в Pinhead:
- Ошибка при сохранении — `useOrdersStore.js` + `StepSummary.jsx`
- Ошибка в визарде — `useStore.js` + компонент шага
- Ошибка авторизации — `useAuthStore.js`
- Ошибка каталогов — `catalogSlice.js` + `lib/catalogs.js`
- Ошибка цен — `utils/pricing.js`

## Шаг 2 — Изолируй причину

Задай себе 3 вопроса:
1. В каком слое баг — UI, store, Supabase или бизнес-логика?
2. Это регрессия (работало раньше) или фича никогда не работала?
3. Воспроизводится всегда или только при определённых условиях?

Проверь тесты:
```bash
cd pinhead-react && npm test -- --run 2>&1 | grep -E "FAIL|Error" | head -20
```

## Шаг 3 — Найди корневую причину

НЕ чини симптом — найди причину.

Паттерны багов в Pinhead:
- **Silent fail** — Supabase возвращает ошибку, catch пустой — добавь `toast.error`
- **Потеря данных** — optimistic update без rollback — верни старое состояние при ошибке
- **Неправильный шаг** — `editItem` переходит не туда — проверь `itemsSlice.js`
- **Старые каталоги** — sessionStorage кэш — очисти через `sessionStorage.clear()`
- **DEV_MODE** — в dev авторизация пропущена, `userId = 'dev'` — не UUID, осторожно с Supabase

## Шаг 4 — Почини минимально

Правило: минимальное изменение которое решает проблему.

- НЕ рефактори попутно
- НЕ меняй файлы не связанные с багом
- Одно изменение = одна причина

После правки:
```bash
cd pinhead-react && npm test -- --run
```

Baseline: **722 теста зелёных**. Если упало — это регрессия, откати и ищи дальше.

## Шаг 5 — Проверь и задокументируй

- Убедись что баг исправлен
- Убедись что смежный функционал не сломан
- Коммит формата: `fix(scope): краткое описание причины и фикса`
