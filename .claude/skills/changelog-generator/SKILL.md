# Changelog Generator — Pinhead Order Studio

Генерирует читаемый changelog из git-коммитов.

## Шаг 1 — Получи коммиты

```bash
# За последнюю сессию (с последнего тега или даты)
git log --oneline --since="2026-04-07" --no-merges

# Или за последние N коммитов
git log --oneline -20 --no-merges

# С датами
git log --pretty=format:"%ad %s" --date=short -20 --no-merges
```

## Шаг 2 — Сгруппируй по типу

Разбей коммиты на категории:

- **Исправления багов** — коммиты начинающиеся с `fix(`
- **Новый функционал** — `feat(`
- **UX улучшения** — `feat(wizard)`, `feat(kanban)`, `feat(mobile)`
- **Производительность** — `perf(`
- **Документация** — `docs(`
- **Рефакторинг** — `refactor(`

## Шаг 3 — Напиши в формате CHANGELOG.md

```markdown
## [Дата] — Сессия X

### Исправления багов
- Заказ больше не теряется при ошибке Supabase (saveOrder возвращает null)
- Обновление заказа: откат при ошибке + null вместо фейкового объекта
- Удаление заказа: подтверждение + ожидание ответа Supabase

### Новый функционал  
- Кнопка «Повторить заказ» — дублирует и открывает в визарде
- Комментарии к заказу (таблица order_comments в Supabase)
- Шаблоны изделий (таблица order_templates в Supabase)
- Пагинация заказов: 50 за раз + «Загрузить ещё»

### UX улучшения
- Визард: 6 → 5 шагов (обработки в аккордеон)
- Индикатор заполненности шагов в прогресс-баре
- Мобильная оптимизация свотчей и таблицы размеров
- Редактирование позиции — сразу на шаг «Дизайн»

### Документация
- PINHEAD-SKILL.md — скилл-файл для Claude
- ACTION-PLAN.md — обновлён статус задач

---
Тестов: 722 ✅ | Коммитов: X
```

## Шаг 4 — Обнови файл

Добавь новый раздел в начало `CHANGELOG.md` (или создай если нет):

```bash
# Проверить есть ли файл
ls CHANGELOG.md

# Если нет — создать
touch CHANGELOG.md
```

## Правила написания

- Пиши для людей, не для разработчиков
- «Заказ больше не теряется» вместо «fix: saveOrder returns null on error»
- Группируй по ценности для пользователя, не по файлам
- Всегда указывай количество тестов в конце
