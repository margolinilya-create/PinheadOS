### 2. Архитектура: разбивка на файлы
Монолитный `index.html` (10 084 строки, 500 КБ) → модульная структура:

**CSS (6 файлов, 2 210 строк):**
- `css/base.css` — токены, ресет, стили печати
- `css/layout.css` — хедер, прогресс-бар, layout, лейблы
- `css/components.css` — гарменты, цвета, формы, вывод
- `css/panels.css` — прогрессивные секции, экспресс-калькулятор, редактор цен
- `css/auth.css` — авторизация, админ-панель
- `css/extras.css` — декоративные элементы, спиннер, SKU-редактор

**JS (13 файлов, 6 851 строка):**
- `js/config.js` — глобальный обработчик ошибок
- `js/auth.js` — Supabase авторизация, облачное сохранение, админка
- `js/state.js` — объект состояния приложения
- `js/data.js` — таблицы цен, ткани, палитры цветов, константы
- `js/draft.js` — автосохранение в localStorage
- `js/navigation.js` — рендер шагов, прогрессивные секции
- `js/ui-garments.js` — UI выбора изделий
- `js/ui-design.js` — свотчи цветов, выбор ткани
- `js/zones.js` — зоны нанесения, размерная сетка
- `js/pricing.js` — движок ценообразования, техники по зонам
- `js/output.js` — мокап, сводка заказа, PDF-генерация
- `js/history.js` — история заказов, восстановление, инициализация
- `js/editors.js` — редакторы цен, SKU-каталог, экспресс-калькулятор

**HTML (1 036 строк):**
- `index.html` — чистый шаблон + ссылки на CSS/JS

PDF-генерация не затронута (собирает CSS инлайн).

### 3. Редизайн Print Preview ТЗ
Полностью переработан визуал документа ТЗ (функция `generatePDF()`):
- **Шрифты** — Barlow Condensed для заголовков, Roboto Condensed для тела
- **Шапка** — логотип PINHEAD + подпись "Order Studio", контакты справа
- **Нумерованные секции** — номер в квадрате 24×24 + заголовок uppercase с чёрной линией
- **Таблицы** — чёрный заголовок с белым текстом, зебра-полосы, жирная итоговая строка
- **Теги опций** — чёрные плашки (tag-active) вместо серых рамок
- **Итого** — блок с рамкой 3px, лейбл слева + сумма справа акцентным цветом (#1D19EA)
- **Панель действий** — чёрная полоса сверху с кнопками "ПЕЧАТЬ/PDF" и "ЗАКРЫТЬ"
- **Поле мессенджер** — добавлено в секцию клиента
- **Футер** — версия + дата формирования (скрыт при печати)
